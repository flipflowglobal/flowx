# NEXUS-ARB v2.0
### Autonomous DeFi Arbitrage Engine

> **Live mainnet flash-loan arbitrage across Uniswap V3, Curve, and Balancer V2 вҖ” powered by PPO, Thompson Sampling, UKF, and CMA-ES.**

---

## What It Does

NEXUS-ARB continuously scans Ethereum mainnet (and Arbitrum One) for profitable arbitrage cycles across three major DEX protocols. When a viable opportunity is found, it borrows capital via Aave V3 flash loans (zero upfront capital required), executes the full swap sequence atomically on-chain, repays the loan in the same transaction, and keeps the profit.

**Key capabilities:**
- Bellman-Ford negative-cycle detection on a live log-price graph (mathematically exact arbitrage detection)
- Monte Carlo profit simulation (500 samples per route, slippage-aware)
- AI cognitive layer: PPO + Thompson Sampling + UKF + CMA-ES with Shapley weight attribution
- Kelly Criterion position sizing with tri-state circuit breaker
- Real-time terminal dashboard with WebSocket live updates
- ARM64-compatible вҖ” runs on Android (UserLAnd Ubuntu), Raspberry Pi, and standard Linux

## Quick Start (10 commands)

```bash
# 1. Clone
git clone https://github.com/yourname/nexus-arb.git
cd nexus-arb

# 2. Run installer (handles Python venv, Node, Hardhat, DB init)
chmod +x setup.sh && ./setup.sh

# 3. Configure
cp .env.example .env
nano .env   # Add ETH_RPC_URL and DEPLOYER_PRIVATE_KEY

# 4. Deploy smart contract to mainnet
npm run deploy:mainnet

# 5. Set deployed address (shown in step 4 output)
# Edit .env: FLASH_RECEIVER_ADDRESS=0xYOUR_DEPLOYED_ADDRESS

# 6. Start dry-run first (no real transactions)
python3 main.py --dry-run

# 7. Watch dashboard
# Open browser: http://localhost:8420

# 8. Review logs
tail -f logs/nexus-arb.log

# 9. If satisfied, stop dry-run
./stop.sh

# 10. Start live
./start.sh
```

## Configuration Reference

All settings live in `.env`. Key parameters:

| Variable | Default | Description |
|---|---|---|
| `ETH_RPC_URL` | required | Primary Ethereum RPC (Alchemy/Infura) |
| `DEPLOYER_PRIVATE_KEY` | required | Wallet that owns deployed contract |
| `FLASH_RECEIVER_ADDRESS` | required | Deployed NexusFlashReceiver address |
| `MIN_PROFIT_USD` | `5.0` | Minimum net profit to execute |
| `MAX_LOAN_USD` | `50000.0` | Maximum flash loan size |
| `MAX_GAS_GWEI` | `50.0` | Abort if gas exceeds this |
| `SLIPPAGE_BPS` | `50` | Slippage tolerance (0.5%) |
| `SCAN_INTERVAL_MS` | `500` | Time between scan cycles |

See `docs/CONFIGURATION.md` for all variables.

## Algorithm Stack

| Component | Algorithm | Reference |
|---|---|---|
| Arbitrage detection | Bellman-Ford on log-price graph | Avellaneda & Stoikov (2008) |
| Profit simulation | Monte Carlo (500 samples) | Wintermute methodology |
| RL action selection | PPO with linear policy | Schulman et al. (2017) |
| Contextual bandit | Thompson Sampling (Gaussian-Gaussian) | Agrawal & Goyal (2012) |
| Spread prediction | Unscented Kalman Filter (OU process) | Julier & Uhlmann (1997) |
| Parameter optimization | CMA-ES | Hansen & Ostermeier (2001) |
| Engine weighting | Shapley Value attribution | Lundberg & Lee (2017) |
| Position sizing | Kelly Criterion (quarter-Kelly) | Kelly (1956) |
| Experience replay | Prioritized (sum-tree PER) | Schaul et al. (2015) |
| RPC selection | EWMA latency-adaptive | MEV searcher standard |
| Gas bidding | EIP-1559 90th-pct fee | MetaMask "fast" methodology |

## Security

- Private key is **only** in `.env` вҖ” never in source code
- Contract has `onlyOwner` + `onlyAavePool` guards
- `Pausable` for emergency stop: call `pause()` on contract
- Circuit breaker limits max drawdown to $200 before halting
- All swaps have explicit `minAmountOut` slippage floors
- See `docs/SECURITY.md` for full threat model

## Troubleshooting

**`ConfigurationError: Required environment variable 'ETH_RPC_URL' is not set`**
вҶ’ Copy `.env.example` to `.env` and fill in your RPC URL.

**`NexusFlashReceiver not deployed at 0x000...`**
вҶ’ Run `npm run deploy:mainnet` and set `FLASH_RECEIVER_ADDRESS` in `.env`.

**`Gas too high: 85.00 gwei > max 50.0 gwei`**
вҶ’ Increase `MAX_GAS_GWEI` in `.env` or wait for gas to drop.

**`CircuitBreaker OPEN`**
вҶ’ 3 consecutive reverts or drawdown exceeded. Auto-resets after 5 minutes.

**Scan cycles return 0 viable routes**
вҶ’ Normal during high-gas / low-volatility periods. Check dashboard for gas gauge.

See `docs/TROUBLESHOOTING.md` for more.
