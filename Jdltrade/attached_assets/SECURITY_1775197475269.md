# NEXUS-ARB v2 — Security Guide

## Private Key Management

**Rules:**
1. Private key lives ONLY in `.env` — never in source files, never in git history
2. `.env` is in `.gitignore` — verify with `git status` before any push
3. Set `.env` permissions: `chmod 600 .env`
4. Use a dedicated trading wallet — never your main wallet
5. Keep only enough ETH for gas (0.05–0.2 ETH). Profits accumulate in the wallet.

**If key is compromised:**
1. Call `contract.pause()` immediately (owner-only function)
2. Rotate to new wallet: deploy new contract, update `.env`
3. Revoke all token approvals on old wallet via revoke.cash

## Smart Contract Security

**Access controls:**
- `onlyOwner` on: `initiateFlashLoan`, `rescueTokens`, `pause`, `unpause`
- `onlyAavePool` on: `executeOperation` (prevents spoofed flash loan callbacks)
- `initiator == address(this)` check inside `executeOperation`

**Slippage protection:**
- Each swap step has explicit `minAmountOut` computed as `quote × (1 - slippage_bps/10000)`
- Contract reverts with `SlippageExceeded(received, minimum)` if any step undershoots

**Profitability guard:**
- Contract reverts with `InsufficientProfit(actual, required)` if final balance < loan + premium
- No partial execution possible — atomic or nothing

**Reentrancy:**
- `ReentrancyGuard` from OpenZeppelin on `executeOperation`
- Checks-Effects-Interactions pattern: approve repayment before external transfer

## Circuit Breaker (Python layer)

Opens (halts execution) when:
- 3 consecutive transaction reverts
- Kelly fraction f* < 0 (negative expected value from recent trade history)
- Rolling drawdown > $200 in last 50 trades

Auto-recovers after 5 minutes via HALF_OPEN probe state.

## RPC Security

- Use authenticated endpoints (Alchemy/Infura API keys) — not public RPCs for mainnet trading
- Never log raw RPC responses containing price data that could be front-run
- WS connections use TLS (wss://) on all configured endpoints

## Maximum Loss Scenarios

| Scenario | Maximum Loss | Mitigation |
|---|---|---|
| Flash loan reverts | Gas cost only (~$5-20) | Circuit breaker opens after 3 |
| Slippage worse than expected | Gas cost only (tx reverts) | minAmountOut floor |
| Smart contract bug | Contract ETH balance | Pausable + rescueTokens |
| Private key stolen | Wallet ETH balance | Dedicated wallet + low balance |
| Gas spike | Gas cost overpay | MAX_GAS_GWEI limit |

Flash loans guarantee: if the arbitrage is unprofitable, the transaction reverts and you only lose gas.

## Audit Checklist

Before mainnet deployment, verify:
- [ ] `executeOperation` caller check passes (`msg.sender == AAVE_POOL`)
- [ ] `initiator` check passes (`initiator == address(this)`)
- [ ] `InsufficientProfit` revert fires on negative-profit route
- [ ] `SlippageExceeded` revert fires when swap returns less than `minAmountOut`
- [ ] `rescueTokens` works for stuck ERC20s
- [ ] `pause()` / `unpause()` blocks / restores flash loan execution
- [ ] No ETH sent to constructor (not payable)
- [ ] Slither and Mythril show no high-severity findings
